# Contributing to Dine Smart

Thanks for your interest in contributing! This document explains how to get the code running locally and how to submit changes.

## Getting started

1. Fork the repository and create a feature branch from `main`.
2. Follow the setup instructions in `README.md` for backend and frontend.

## Code style and tests

- Python: follow PEP8. We recommend using `black` for formatting.
- JavaScript/React: use the existing ESLint configuration; run `npm run lint`.
- Add unit tests for new backend functionality using Django's test framework.

## Pull requests

- Keep PRs small and focused.
- Include tests and update documentation when behavior changes.
- Describe the problem, your approach, and any migration steps in the PR description.

## Reporting issues

Open an issue in the tracker with a reproducible description, steps, and expected vs actual behavior.
